This manual describes the features and customization options for the Emacs Lisp package =elfeed-ai=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

elfeed-ai adds AI-powered content curation to [[https://github.com/skeeto/elfeed][elfeed]], the Emacs
feed reader.  It uses [[https://github.com/karthink/gptel][gptel]] to score each entry for relevance against
a natural-language interest profile, tags entries that meet a
configurable threshold, displays scores as a column in the search
buffer, and injects AI-generated summaries into the show buffer.

The package solves the problem of information overload: rather than
manually scanning every entry in your feeds, you describe your
interests once, and elfeed-ai continuously evaluates new entries and
surfaces only the content that matters to you.  Entries that score
above a configurable relevance threshold are tagged =ai-relevant=,
which you can use in elfeed's standard search filter syntax.

elfeed-ai integrates with elfeed through three mechanisms:

- *Automatic scoring* via =elfeed-new-entry-hook=.  When new entries
  arrive during an elfeed update, they are queued and scored
  asynchronously one at a time.  Each scored entry receives an AI
  relevance score and a brief summary, stored as =elfeed-meta=
  properties.

- *Search buffer column*.  A custom
  =elfeed-search-print-entry-function= adds a score column between
  the date and title, with color-coded faces for high and low scores.

- *Show buffer summary*.  After-advice on =elfeed-show-refresh=
  injects the AI-generated summary above the original content, giving
  you a quick overview before reading the full entry.

A daily token budget prevents runaway API costs.  Token usage is
estimated heuristically and tracked in a small persistence file.

* Quick start
:PROPERTIES:
:CUSTOM_ID: h:quick-start
:END:

#+begin_src emacs-lisp
(require 'elfeed-ai)

(setq elfeed-ai-interest-profile
      "AI safety, Emacs, functional programming, philosophy of mind,
       effective altruism, longevity research")

(elfeed-ai-mode 1)
#+end_src

New entries are scored automatically.  Use =+ai-relevant= in your
elfeed search filter to see only curated content:

#+begin_example
+ai-relevant +unread
#+end_example

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Interest profile
:PROPERTIES:
:CUSTOM_ID: h:interest-profile
:END:

#+vindex: elfeed-ai-interest-profile
The user option ~elfeed-ai-interest-profile~ is a free-form string
describing your interests in natural language.  This string is included
verbatim in every AI scoring prompt.  The more specific and descriptive
it is, the better the AI can judge relevance.  When empty, scoring is
skipped entirely.

** Relevance threshold
:PROPERTIES:
:CUSTOM_ID: h:relevance-threshold
:END:

#+vindex: elfeed-ai-relevance-threshold
The user option ~elfeed-ai-relevance-threshold~ sets the minimum AI
score (a float between 0.0 and 1.0) for an entry to receive the
~elfeed-ai-score-tag~ tag.  The default value is ~0.5~.

** Token budget
:PROPERTIES:
:CUSTOM_ID: h:token-budget
:END:

#+vindex: elfeed-ai-daily-token-budget
#+vindex: elfeed-ai-budget-reset-hour
#+vindex: elfeed-ai-budget-file
The user option ~elfeed-ai-daily-token-budget~ caps the estimated
number of tokens consumed per day.  The default is ~100000~.  Token
usage is estimated as one token per four characters.

The user option ~elfeed-ai-budget-reset-hour~ specifies the hour
(0--23) when the daily budget resets.  The default is ~0~ (midnight).

The user option ~elfeed-ai-budget-file~ specifies the file used to
persist daily token usage across sessions.

** Content length
:PROPERTIES:
:CUSTOM_ID: h:content-length
:END:

#+vindex: elfeed-ai-max-content-length
The user option ~elfeed-ai-max-content-length~ sets the maximum number
of characters of entry content sent to the AI.  Longer content is
truncated.  The default is ~4000~.

** Tags
:PROPERTIES:
:CUSTOM_ID: h:tags
:END:

#+vindex: elfeed-ai-score-tag
#+vindex: elfeed-ai-scored-tag
The user option ~elfeed-ai-score-tag~ is the tag added to entries that
score above the threshold.  The default is ~ai-relevant~.

The user option ~elfeed-ai-scored-tag~ is the tag added to all entries
after scoring, regardless of their score.  The default is ~ai-scored~.

** Auto-scoring
:PROPERTIES:
:CUSTOM_ID: h:auto-scoring
:END:

#+vindex: elfeed-ai-auto-score
The user option ~elfeed-ai-auto-score~ controls whether new entries
are automatically queued for scoring.  The default is ~t~.  Set to
~nil~ if you prefer to score entries manually.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Enabling the mode
:PROPERTIES:
:CUSTOM_ID: h:enabling-the-mode
:END:

#+findex: elfeed-ai-mode
The command ~elfeed-ai-mode~ is a global minor mode that activates all
elfeed-ai integrations: automatic scoring of new entries, the score
column in the search buffer, and summary injection in the show buffer.

** Manual scoring
:PROPERTIES:
:CUSTOM_ID: h:manual-scoring
:END:

#+findex: elfeed-ai-score-entry-at-point
#+findex: elfeed-ai-score-selected
#+findex: elfeed-ai-score-unscored
The command ~elfeed-ai-score-entry-at-point~ scores the entry at point
in either the search or show buffer.

The command ~elfeed-ai-score-selected~ queues all selected entries in
the search buffer for scoring.

The command ~elfeed-ai-score-unscored~ walks the entire elfeed database
and queues all entries that lack the ~elfeed-ai-scored-tag~ tag.

** Budget status
:PROPERTIES:
:CUSTOM_ID: h:budget-status
:END:

#+findex: elfeed-ai-budget-status
The command ~elfeed-ai-budget-status~ displays the current daily token
usage in the echo area.

* Faces
:PROPERTIES:
:CUSTOM_ID: h:faces
:END:

- ~elfeed-ai-score-face~ --- default face for scores in the search
  buffer.
- ~elfeed-ai-score-high-face~ --- face for scores >= 0.7 (inherits
  from ~success~).
- ~elfeed-ai-score-low-face~ --- face for scores <= 0.3 (inherits from
  ~shadow~).
- ~elfeed-ai-summary-heading-face~ --- face for the "AI Summary"
  heading in the show buffer.
